Notes
  • MCP (Model Context Protocol) standardizes how the LLM application discovers and invokes tools/resources from external systems like SAP.
  • Use least-privilege users and secure all connections.
  • This setup works with any LLM that supports tool calling.
Flow Summary
  1. User asks a question in natural language.
  2. LLM decides to call a tool/resource via MCP.
  3. MCP Server calls SAP using appropriate API.
  4. SAP processes the request.
  5. (implicit) Cross-cutting services apply (auth, logging, etc.).
  6. SAP returns data to MCP Server.
  7. MCP Server returns result to MCP Host.
  8. LLM generates the final response to the user.
